MsQuic Performance Dashboard

This page contains performance related data for MsQuic and is automatically generated via our CI pipeline.

CPU Limited Tests

The data is generated for several different build configurations and scenarios. Generally, the specific on each scenario are described here. Each data set is a combination of OS/platform (Windows Kernel, Windows User, or Linux User) and the TLS library (Schannel, OpenSSL) used. The tests are run on two physical machines connected by a 50 Gbps switch (with 40 Gbps NICs), so the tests are practically limited by the CPU.

Single Connection Throughput

This test consists of transferring data over a single QUIC connection, single stream in one direction as fast as possible.

Requests per Second

This test consists creating many parallel connections and then sending requests (and waiting for the responses) as fast as possible.

Handshake per Second

This test consists creating new connections (handshakes) as fast as possible.
